# 该项目是关系抽取加实体抽取

启动 app.py 文件，实现离线部分的关系抽取，实体抽取，送入 neo4j 数据库当中。

启动 query.py 文件，实现在线部分的关系抽取，实体抽取，进入 neo4j 数据库查询对应的实体和对应的关系。

最终查询出来的结果如下所示：

```
===== 查询实体 =====
{'labels': ['地点'], 'name': '北京市'}
{'labels': ['时间'], 'name': '2022年'}

===== 查询关系 =====
{'from': {'labels': ['比赛事件'], 'name': '2022年北京马拉松在北京市举办'}, 'relation': '发生时间', 'to': {'labels': ['时间'], 'name': '2022年'}}
{'from': {'labels': ['比赛事件'], 'name': '2022年北京马拉松在北京市举办'}, 'relation': '发生地点', 'to': {'labels': ['地点'], 'name': '北京市'}}

(uie_env) D:\Productmap>

````


# 文件说明

EntityExtractor 文件夹是基于 UIE 推理服务的项目，进入这个目录后，是基于 BERT + UIE 的推理服务。

SpellingCorrection 文件夹是基于 BERT 的错字识别的项目，进入这个目录后，是基于 BERT 的错字识别的项目，错字识别服务。

# 后期优化点

在查询侧实现多层次检索，初步检索，外加基于 Neo4j 的检索，或基于正则的检索不到后，再次进入 Neo4j 进行检索。
